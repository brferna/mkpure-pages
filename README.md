# mkpure-pages

一个用于归档和发布多个独立 HTML 页面的仓库。

## 项目简介

本仓库主要用于存放各类独立的 HTML 页面文件，这些页面可以是静态网页、交互式演示、工具页面或信息展示页等。仓库本身不针对任何特定域名或网站，仅作为页面文件的归档与托管空间，方便通过 GitHub Pages 或其他静态托管服务进行访问。

## 目录结构

```
mkpure-pages/
├── README.md          # 本文件
└── pages/             # 存放所有 HTML 页面的目录
    ├── example.html   # 示例页面
    └── ...
```

- `pages/`：所有独立 HTML 页面均放置在此目录下，每个页面为一个独立的 `.html` 文件。
- 页面文件名应简洁、语义化，便于识别与链接访问。

## 页面归档说明

- 每个 HTML 页面均为独立完整文件，包含所需的样式与脚本（或引用外部资源）。
- 页面之间无依赖关系，可单独访问。
- 归档的页面类型包括但不限于：
  - 信息展示页
  - 工具页面
  - 交互式演示
  - 临时或测试页面
- 页面内容应遵循互联网通用规范，不包含违法违规、侵权或恶意内容。

## 维护说明

- 欢迎通过 Pull Request 提交新的 HTML 页面，或对现有页面进行改进。
- 提交新页面时，请确保文件命名清晰，并简要说明页面用途。
- 不建议在页面中添加跟踪、广告或外部第三方脚本（除非必要且明确说明）。
- 仓库维护者会定期检查页面可用性，并可能移除长期无人维护或已失效的页面。
